# Route Map
- **/match** → `frontend/app/match/page.tsx` loads `MatchPageClient` (`frontend/app/match/MatchPageClient.tsx`) for the one-question-at-a-time intake, mounts `ConsentModal` (`frontend/components/ConsentModal.tsx`), keeps answers in local storage, and on completion routes people to `/trials`.
- **/match/wizard** → `frontend/app/match/wizard/page.tsx` renders the legacy multi-panel `MatchWizard` (`frontend/app/match/wizard/MatchWizard.tsx`) with optional chat assist and preview sidebar before linking to trial results.
- **/trial/[nct_id]/screen** → `frontend/app/trial/[nct_id]/screen/page.tsx` fetches trial data, hydrates `TrialScreenClient` (`frontend/app/trial/[nct_id]/screen/TrialScreenClient.tsx`), and swaps between `Screener` (`frontend/components/Screener.tsx`), `ScreenResult` (`frontend/components/ScreenResult.tsx`), and downstream `LeadForm` (`frontend/components/LeadForm.tsx`).
- **/trials** → `frontend/app/trials/page.tsx` executes the search/match query, renders `TrialsMatchedClient`, and surfaces `TrialCard` (`frontend/components/trials/TrialCard.tsx`) with the explainability `ScoreBreakdownSheet` (`frontend/components/trials/ScoreBreakdownSheet.tsx`) plus shortlist tooling.
